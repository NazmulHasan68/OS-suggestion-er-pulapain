অপারেটিং সিস্টেমের বিবর্তন ও আর্কিটেকচার
Evolution & Architecture of Operating System
Q
বাং অপারেটিং সিস্টেম কী? এটি কোন ভূমিকা পালন করে?
EN What is an Operating System? What role does it play?
▾
✦ বাংলা উত্তর

অপারেটিং সিস্টেম (OS) হলো একটি সিস্টেম সফটওয়্যার যা ব্যবহারকারী এবং কম্পিউটার হার্ডওয়্যারের মধ্যে সেতু হিসেবে কাজ করে। OS ছাড়া কম্পিউটার কোনো প্রোগ্রাম চালাতে পারে না। ব্যবহারকারীর কমান্ড (ক্লিক, টাইপ) OS হার্ডওয়্যার নির্দেশে রূপান্তর করে।

সংক্ষেপে: User → Operating System → Hardware — এই তিন স্তর মিলে সিস্টেম গঠিত।
✦ English Answer

An Operating System (OS) is system software that acts as a bridge between the user and the computer hardware. Without an OS, a computer cannot run any program. It translates user commands (clicks, keystrokes) into hardware instructions.

In short: User → Operating System → Hardware — these three layers form a complete computing system.
Q
বাং অপারেটিং সিস্টেমের পাঁচটি প্রধান কাজ কী কী?
EN What are the five main functions of an Operating System?
▾
✦ বাংলা উত্তর
  1. প্রসেস ম্যানেজমেন্ট: কোন প্রোগ্রাম কখন চলবে তা নিয়ন্ত্রণ করে।
  2. মেমোরি ম্যানেজমেন্ট: প্রোগ্রামকে মেমোরি বরাদ্দ দেয় এবং ব্যবহার শেষে ফিরিয়ে নেয়।
  3. ফাইল সিস্টেম ম্যানেজমেন্ট: ডিস্কে ফাইল সংরক্ষণ ও সংগঠিত করে।
  4. ডিভাইস ম্যানেজমেন্ট: কীবোর্ড, প্রিন্টার, মাউস পরিচালনা করে।
  5. নিরাপত্তা ও অ্যাক্সেস নিয়ন্ত্রণ: অননুমোদিত ব্যবহারকারীদের থেকে সিস্টেম সুরক্ষিত রাখে।
✦ English Answer
  1. Process Management: Controls which program runs and when.
  2. Memory Management: Allocates memory to programs and reclaims it after use.
  3. File System Management: Stores and organizes files on disk.
  4. Device Management: Manages hardware like keyboard, printer, and mouse.
  5. Security & Access Control: Protects the system from unauthorized users.
Q
বাং অপারেটিং সিস্টেমের ৫টি প্রজন্ম সংক্ষেপে ব্যাখ্যা করো।
EN Briefly explain the 5 generations of Operating Systems.
▾
✦ বাংলা উত্তর
  1. ১ম প্রজন্ম (১৯৪৫–৫৫): কোনো OS ছিল না। ভ্যাকুয়াম টিউব ও পাঞ্চড কার্ড। উদাহরণ: ENIAC।
  2. ২য় প্রজন্ম (১৯৫৫–৬৫): ব্যাচ প্রসেসিং। ট্রানজিস্টর ব্যবহার। উদাহরণ: IBM OS/360।
  3. ৩য় প্রজন্ম (১৯৬৫–৮০): মাল্টিপ্রোগ্রামিং ও টাইম-শেয়ারিং। UNIX জন্ম নেয় (১৯৬৯)।
  4. ৪র্থ প্রজন্ম (১৯৮০–বর্তমান): পার্সোনাল কম্পিউটার ও GUI। উদাহরণ: Windows, macOS।
  5. ৫ম প্রজন্ম (বর্তমান–ভবিষ্যৎ): মোবাইল, ক্লাউড ও AI OS। উদাহরণ: Android, iOS, Chrome OS।
✦ English Answer
  1. 1st Gen (1945–55): No OS. Vacuum tubes and punched cards. Example: ENIAC.
  2. 2nd Gen (1955–65): Batch processing. Transistors used. Example: IBM OS/360.
  3. 3rd Gen (1965–80): Multiprogramming & time-sharing. UNIX born in 1969.
  4. 4th Gen (1980–present): Personal computers and GUI. Example: Windows, macOS.
  5. 5th Gen (present–future): Mobile, cloud & AI OS. Example: Android, iOS, Chrome OS.
Q
বাং মনোলিথিক ও মাইক্রোকার্নেল আর্কিটেকচারের পার্থক্য লেখো।
EN Differentiate between Monolithic and Microkernel architecture.
▾
✦ বাংলা উত্তর
  • মনোলিথিক: সমস্ত OS সার্ভিস একটি বড় কার্নেল ব্লকে চলে। খুব দ্রুত। কিন্তু একটি বাগ পুরো সিস্টেম ক্র্যাশ করতে পারে। উদাহরণ: Linux, MS-DOS।
  • মাইক্রোকার্নেল: কার্নেলে শুধু IPC, মেমোরি ও CPU শিডিউলিং থাকে। বাকি সার্ভিস ইউজার-স্পেসে। নিরাপদ কিন্তু তুলনামূলক ধীর। উদাহরণ: MINIX, QNX।
মনে রাখো: Monolithic = গতি, Microkernel = নিরাপত্তা, Hybrid = উভয়ের সেরা।
✦ English Answer
  • Monolithic: All OS services run inside one large kernel block. Very fast but a bug in one part can crash the whole system. Example: Linux, MS-DOS.
  • Microkernel: Only IPC, memory management, and CPU scheduling stay in the kernel. Other services run in user space. Highly secure but slower due to message passing. Example: MINIX, QNX.
Remember: Monolithic = speed, Microkernel = security, Hybrid = best of both.
Q
বাং লেয়ারড ও মডুলার আর্কিটেকচার কীভাবে কাজ করে?
EN How do Layered and Modular architectures work?
▾
✦ বাংলা উত্তর

লেয়ারড: OS-কে নির্দিষ্ট নম্বরযুক্ত স্তরে ভাগ করা হয়। প্রতিটি স্তর শুধুমাত্র তার নিচের স্তরের সার্ভিস ব্যবহার করে। ডিবাগ সহজ কিন্তু ধীর। উদাহরণ: THE OS (Dijkstra, ১৯৬৮)।

মডুলার: OS আলাদা স্বাধীন মডিউল দিয়ে তৈরি। সিস্টেম চলার সময়েই মডিউল যোগ বা বাদ দেওয়া যায়। খুব নমনীয়। উদাহরণ: Modern Linux, Solaris।

✦ English Answer

Layered: The OS is divided into numbered layers. Each layer only uses services from the layer directly below it. Easy to debug but slower due to multiple layer traversal. Example: THE OS (Dijkstra, 1968).

Modular: The OS is built from separate, independent modules — each handling one task. Modules can be added or removed while the system runs. Very flexible. Example: Modern Linux, Solaris.

Q
বাং হাইব্রিড আর্কিটেকচার কী এবং কোন OS এটি ব্যবহার করে?
EN What is Hybrid architecture and which OS uses it?
▾
✦ বাংলা উত্তর

হাইব্রিড আর্কিটেকচার হলো মনোলিথিক ও মাইক্রোকার্নেলের সমন্বয়। মূল সার্ভিসগুলো (Process Manager, Memory Manager, Core Drivers) কার্নেল স্পেসে — গতির জন্য। অন্যান্য সার্ভিস (GUI, Network Stack) ইউজার স্পেসে — নিরাপত্তার জন্য।

উদাহরণ: Windows NT/10/11, macOS (XNU Kernel), Android।
✦ English Answer

Hybrid architecture combines features of Monolithic and Microkernel designs. Core services (Process Manager, Memory Manager, Core Drivers) run in kernel space for speed; other services (GUI, Network Stack, Security) run in user space for safety.

Examples: Windows NT/10/11, macOS (XNU Kernel), Android.

ফাইল সিস্টেম ডিজাইন ও বাস্তবায়ন
File System Design and Implementation
Q
বাং ফাইল সিস্টেম কী? এটি কেন প্রয়োজন?
EN What is a File System? Why is it needed?
▾
✦ বাংলা উত্তর

ফাইল সিস্টেম হলো OS-এর এমন একটি অংশ যা HDD/SSD থেকে ডেটা কীভাবে সংরক্ষণ ও উদ্ধার হবে তা পরিচালনা করে। এটি কাঁচা বিটগুলোকে ফাইল ও ডিরেক্টরিতে সংগঠিত করে।

ফাইল সিস্টেম ছাড়া: কোনো ফাইল কোথায় শুরু বা শেষ হয় বোঝা যায় না এবং ব্যবহারকারীদের মধ্যে কোনো নিরাপত্তা থাকে না।

উপমা: ফাইল সিস্টেম = লাইব্রেরিয়ান যে প্রতিটি বইকে সঠিক তাকে সাজিয়ে রাখে।
✦ English Answer

A file system is the part of the OS that manages how data is stored and retrieved from storage devices (HDD, SSD). It organizes raw bits into meaningful files and directories.

Without a file system: there is no way to know where a file starts or ends, and there is no security or sharing between users.

Analogy: File system = a librarian giving every book a proper shelf address.
Q
বাং ফাইল বরাদ্দের তিনটি পদ্ধতি বিস্তারিত ব্যাখ্যা করো।
EN Explain the three file allocation methods in detail.
▾
✦ বাংলা উত্তর
  1. Contiguous (ধারাবাহিক): ফাইল পরপর ডিস্ক ব্লকে থাকে। দ্রুত sequential access। কিন্তু external fragmentation ও ফাইল বড় করা কঠিন।
  2. Linked (সংযুক্ত): প্রতিটি ব্লকে পরবর্তী ব্লকের pointer। Fragmentation নেই, ফাইল সহজে বাড়ানো যায়। কিন্তু random access ধীর।
  3. Indexed (সূচিভিত্তিক): একটি index block-এ সব ডেটা ব্লকের ঠিকানা। দ্রুত random access। তবে index block নিজেই জায়গা নেয়।
✦ English Answer
  1. Contiguous Allocation: File stored in consecutive disk blocks. Fast sequential access. But causes external fragmentation and hard to grow files.
  2. Linked Allocation: Each block has a pointer to the next block. No fragmentation and easy to grow. But slow random access and pointer overhead per block.
  3. Indexed Allocation: One index block holds all data block addresses. Fast random access and no fragmentation. But the index block itself takes space; multi-level index needed for very large files.
Q
বাং ফ্রি স্পেস ম্যানেজমেন্টের দুটি পদ্ধতি কী কী?
EN What are the two methods of Free Space Management?
▾
✦ বাংলা উত্তর
  1. Bit Map: প্রতিটি ডিস্ক ব্লককে একটি বিট দিয়ে উপস্থাপন করা হয়। 0 = ফ্রি, 1 = ব্যবহৃত। দ্রুত স্ক্যান। সহজ ও কম্প্যাক্ট।
  2. Free List: ফ্রি ব্লকগুলো একটি Linked List-এ থাকে। ফ্রি ব্লক যোগ ও বাদ দেওয়া সহজ। তবে বড় ডিস্কে traversal ধীর।
✦ English Answer
  1. Bit Map: Each disk block is represented by one bit: 0 = free, 1 = used. Fast scan to find free blocks. Simple and compact.
  2. Free List: Free blocks are linked together — each free block points to the next. Easy to add/remove free blocks. But traversal can be slow for large disks.
Q
বাং ফাইল মেটাডেটা কী? কোন তথ্যগুলো মেটাডেটায় থাকে?
EN What is File Metadata? What information does it contain?
▾
✦ বাংলা উত্তর

মেটাডেটা হলো "ডেটা সম্পর্কে ডেটা" — ফাইলের প্রকৃত বিষয়বস্তু নয়, বরং ফাইলের তথ্য।

  • ফাইলের নাম, আকার, ধরন
  • তৈরি ও পরিবর্তনের তারিখ
  • মালিক ও অনুমতি (Read/Write/Execute)
  • ডেটা ব্লকের ডিস্কে অবস্থান
✦ English Answer

Metadata means "data about data" — not the actual file content, but information about the file.

  • File name, size, and type
  • Created and modified date
  • Owner and permissions (Read/Write/Execute)
  • Location of data blocks on disk
Q
বাং NTFS, ext4 ও FAT32 ফাইল সিস্টেমের মধ্যে পার্থক্য কী?
EN What are the differences between NTFS, ext4, and FAT32 file systems?
▾
✦ বাংলা উত্তর
  • NTFS (Windows): অনুমতি, বড় ফাইল ও journaling সমর্থন করে।
  • ext4 (Linux): দ্রুত, journaling সমর্থিত, ব্যাপকভাবে ব্যবহৃত।
  • FAT32 (USB Drive): সহজ, সব OS-এ সামঞ্জস্যপূর্ণ, তবে 4GB-এর বেশি ফাইল সমর্থন করে না।
✦ English Answer
  • NTFS (Windows): Supports permissions, large files, and journaling for reliability.
  • ext4 (Linux): Fast, supports journaling, widely used in Linux systems.
  • FAT32 (USB Drives): Simple, highly compatible across all OS, but cannot store files larger than 4GB.

প্রসেস ও থ্রেড
Processes and Threads
Q
বাং প্রসেস কী? প্রসেসের বিভিন্ন অবস্থাগুলো বর্ণনা করো।
EN What is a Process? Describe the different process states.
▾
✦ বাংলা উত্তর

প্রসেস হলো একটি চলমান প্রোগ্রাম যার নিজস্ব মেমোরি স্পেস ও রিসোর্স রয়েছে (Code, Data, Stack, Heap)।

  1. New: প্রসেস তৈরি হয়েছে।
  2. Ready: CPU পাওয়ার জন্য অপেক্ষা করছে।
  3. Running: CPU-তে সক্রিয়ভাবে চলছে।
  4. Waiting: I/O বা কোনো ঘটনার জন্য অপেক্ষা করছে।
  5. Terminated: সম্পন্ন হয়েছে।
রূপান্তর: Ready→Running (CPU পেলে), Running→Waiting (I/O লাগলে), Waiting→Ready (I/O শেষে)।
✦ English Answer

A process is a program in execution with its own memory space and resources (Code, Data, Stack, Heap).

  1. New: The process is being created.
  2. Ready: Waiting to be assigned the CPU.
  3. Running: Currently executing on the CPU.
  4. Waiting: Waiting for I/O or an event.
  5. Terminated: Finished execution.
Transitions: Ready→Running (CPU allocated), Running→Waiting (I/O needed), Waiting→Ready (I/O complete).
Q
বাং PCB (Process Control Block) কী? এতে কী কী তথ্য থাকে?
EN What is a PCB (Process Control Block)? What information does it store?
▾
✦ বাংলা উত্তর

PCB হলো OS-এর পরিচালনার জন্য একটি প্রসেস সম্পর্কে সংরক্ষিত সব তথ্যের সমষ্টি।

  • Process ID (PID): অনন্য শনাক্তকারী
  • State: প্রসেসের বর্তমান অবস্থা
  • CPU Registers: Context switching-এর জন্য রেজিস্টার মান
  • Memory Info: বেস ও লিমিট ঠিকানা
  • Priority: শিডিউলিং অগ্রাধিকার
  • I/O Status: খোলা ফাইল ও ডিভাইস
✦ English Answer

A PCB stores all information the OS needs to manage a process.

  • Process ID (PID): Unique identifier
  • State: Current process state
  • CPU Registers: Register values for context switching
  • Memory Info: Base and limit addresses
  • Priority: Scheduling priority level
  • I/O Status: Open files and devices
Q
বাং প্রসেস ও থ্রেডের মধ্যে পার্থক্য কী?
EN What are the differences between a Process and a Thread?
▾
✦ বাংলা উত্তর
  • মেমোরি: প্রসেসের আলাদা মেমোরি; থ্রেড মেমোরি শেয়ার করে।
  • গতি: প্রসেস তৈরি ধীর; থ্রেড তৈরি দ্রুত।
  • Context Switch: প্রসেসে ব্যয়বহুল; থ্রেডে সস্তা।
  • যোগাযোগ: প্রসেসে IPC লাগে; থ্রেডে শেয়ারড মেমোরি।
  • ওভারহেড: প্রসেসে বেশি; থ্রেডে কম।
✦ English Answer
  • Memory: Processes have separate memory; threads share memory.
  • Speed: Processes are slower to create; threads are faster.
  • Context Switch: Expensive for processes; cheap for threads.
  • Communication: Processes need IPC; threads use shared memory.
  • Overhead: Processes have high overhead; threads have low overhead.
Q
বাং মাল্টিথ্রেডিং কী? এর সুবিধা ও অসুবিধাগুলো কী?
EN What is Multithreading? What are its advantages and disadvantages?
▾
✦ বাংলা উত্তর

মাল্টিথ্রেডিং হলো একটি প্রসেসের মধ্যে একাধিক থ্রেড ব্যবহার করে একই সময়ে একাধিক কাজ সম্পন্ন করা।

সুবিধা: দ্রুত সম্পাদন, উন্নত CPU ব্যবহার, রেসপন্সিভনেস, রিসোর্স শেয়ারিং, মাল্টিকোরে স্কেলেবিলিটি।

অসুবিধা: জটিলতা, Race Condition, Deadlock, ডিবাগ কঠিন, সিঙ্ক্রোনাইজেশন ওভারহেড।

✦ English Answer

Multithreading means using multiple threads within a single process to achieve concurrent execution.

Advantages: Faster execution, better CPU utilization, improved responsiveness, resource sharing, scalability on multicore systems.

Disadvantages: Increased complexity, race conditions, deadlock risk, difficult debugging, synchronization overhead.

Q
বাং Amdahl's Law কী? উদাহরণসহ ব্যাখ্যা করো।
EN What is Amdahl's Law? Explain with an example.
▾
✦ বাংলা উত্তর

Amdahl's Law দেখায় N সংখ্যক থ্রেড দিয়ে একটি প্রোগ্রাম কতটুকু দ্রুত হতে পারে।

সূত্র: S = 1 / [(1 − P) + (P / N)] যেখানে S = স্পিডআপ, P = প্যারালেলাইজযোগ্য অংশ, N = কোর সংখ্যা।

উদাহরণ: P=0.8, N=4 হলে S = 1/[0.2+0.2] = 2.5x দ্রুত। অসীম কোরেও সর্বোচ্চ = 1/(1-P) = 5x।
✦ English Answer

Amdahl's Law shows the theoretical speedup of a program using N threads/cores.

Formula: S = 1 / [(1 − P) + (P / N)] where S = Speedup, P = parallelizable fraction, N = number of cores.

Example: P=0.8, N=4 → S = 1/[0.2+0.2] = 2.5x faster. Even with infinite cores, max speedup = 1/(1-P) = 5x.
Q
বাং Mutex, Semaphore ও Monitor-এর মধ্যে পার্থক্য লেখো।
EN Differentiate between Mutex, Semaphore, and Monitor.
▾
✦ বাংলা উত্তর
  • Mutex: বাইনারি লক। একবারে একটি থ্রেড। Deadlock-এর ঝুঁকি।
  • Semaphore: কাউন্টার-ভিত্তিক। Wait/Signal অপারেশন। সাধারণ সিঙ্ক্রোনাইজেশনের জন্য।
  • Monitor: উচ্চ-স্তরের অ্যাবস্ট্রাকশন। Condition variable সমর্থন। Java-তে ব্যবহৃত।
✦ English Answer
  • Mutex: Binary lock — only one thread at a time. Risk of deadlock.
  • Semaphore: Counter-based with Wait/Signal operations. Used for general synchronization.
  • Monitor: High-level abstraction with condition variables. Supported in languages like Java.

CPU শিডিউলিং অ্যালগরিদম
CPU Scheduling Algorithm — Comparative Analysis
Q
বাং CPU শিডিউলিং কী? মূল সূত্রগুলো কী?
EN What is CPU Scheduling? What are the key formulas?
▾
✦ বাংলা উত্তর

CPU শিডিউলিং হলো OS-এর সেই প্রক্রিয়া যেখানে কোন প্রসেস কখন CPU পাবে তা নির্ধারণ করা হয়। লক্ষ্য: CPU ব্যবহার সর্বাধিক ও Waiting Time ন্যূনতম করা।

  • TAT (Turnaround Time) = CT − AT
  • WT (Waiting Time) = TAT − BT
✦ English Answer

CPU Scheduling is the OS process of deciding which process gets the CPU and for how long. Goal: maximize CPU utilization, minimize waiting and turnaround time.

  • TAT (Turnaround Time) = CT − AT (Completion Time − Arrival Time)
  • WT (Waiting Time) = TAT − BT (Turnaround Time − Burst Time)
Q
বাং FCFS অ্যালগরিদম কী? Convoy Effect কী?
EN What is the FCFS algorithm? What is the Convoy Effect?
▾
✦ বাংলা উত্তর

FCFS: যে প্রসেস আগে আসে সে আগে CPU পায়। Non-preemptive — একবার শুরু হলে শেষ পর্যন্ত চলে।

Convoy Effect: একটি বড় প্রসেসের পিছনে ছোট প্রসেসগুলো দীর্ঘ সময় আটকে যায়। উদাহরণের গড়: Avg TAT = 8.33ms, Avg WT = 4.33ms।

✦ English Answer

FCFS (First Come First Served): The process that arrives first gets the CPU first. Non-preemptive — once started, runs to completion.

Convoy Effect: Short processes get stuck waiting behind a long-running process, increasing average waiting time. Example averages: Avg TAT = 8.33ms, Avg WT = 4.33ms.

Q
বাং SJF ও SRTF অ্যালগরিদমের পার্থক্য লেখো।
EN Differentiate between SJF and SRTF scheduling algorithms.
▾
✦ বাংলা উত্তর
  • SJF: Non-preemptive। সবচেয়ে কম Burst Time যার সে আগে। একবার শুরু হলে বাধা দেওয়া যায় না। Avg WT = 3.67ms।
  • SRTF: SJF-এর Preemptive সংস্করণ। নতুন প্রসেস এলে বর্তমান বাকি সময়ের সাথে তুলনা করে। কম BT হলে চলমান প্রসেস থামিয়ে নতুনটি শুরু।
উভয়তেই Starvation-এর ঝুঁকি — বড় প্রসেস CPU নাও পেতে পারে।
✦ English Answer
  • SJF (Shortest Job First): Non-preemptive. Among available processes, picks the shortest burst time. Once started, runs to completion. Avg WT = 3.67ms.
  • SRTF (Shortest Remaining Time First): Preemptive version of SJF. On every new arrival, if new BT < remaining time of running process, preempt immediately.
Both risk Starvation — longer jobs may never get CPU time.
Q
বাং Round Robin অ্যালগরিদম কী? এর মূল বৈশিষ্ট্য কী?
EN What is the Round Robin algorithm? What are its key features?
▾
✦ বাংলা উত্তর

প্রতিটি প্রসেসকে একটি নির্দিষ্ট সময় (Time Quantum/TQ) দেওয়া হয়। TQ শেষে প্রসেস Queue-এর শেষে যায়।

  • Preemptive — সবার জন্য ন্যায্য।
  • কোনো প্রসেস (n−1)×TQ-এর বেশি অপেক্ষা করে না।
  • Time-sharing ও interactive সিস্টেমের জন্য সেরা।
  • কোনো Starvation নেই।
TQ=2ms উদাহরণে: Avg TAT = 8.33ms, Avg WT = 5.00ms — Starvation নেই।
✦ English Answer

Every process gets a fixed time slice (Time Quantum/TQ). After TQ expires, the process moves to the end of the queue.

  • Preemptive — fairness guaranteed for all processes.
  • No process waits longer than (n−1)×TQ.
  • Best suited for time-sharing and interactive systems.
  • No starvation.
With TQ=2ms example: Avg TAT = 8.33ms, Avg WT = 5.00ms — zero starvation.
Q
বাং Starvation কী? Aging দিয়ে কীভাবে সমাধান করা হয়?
EN What is Starvation? How does Aging solve it?
▾
✦ বাংলা উত্তর

Starvation: কম অগ্রাধিকারের প্রসেস কখনো CPU পায় না কারণ উচ্চ অগ্রাধিকারের প্রসেসগুলো বারবার আসে। SJF, Priority ও SRTF-এ ঘটে।

Aging: অপেক্ষারত প্রসেসের অগ্রাধিকার ধীরে ধীরে বাড়ানো হয়। যত বেশি অপেক্ষা, তত বেশি অগ্রাধিকার — একপর্যায়ে CPU পায়। MLFQ-তে ব্যবহৃত।

✦ English Answer

Starvation: A low-priority process never gets the CPU because higher-priority processes keep arriving. Occurs in SJF, Priority, and SRTF.

Aging: Gradually increase the priority of waiting processes. The longer a process waits, the higher it climbs — eventually gets the CPU. Used in MLFQ.

Q
বাং MLQ ও MLFQ-এর পার্থক্য কী? Linux কোনটি ব্যবহার করে?
EN What is the difference between MLQ and MLFQ? Which does Linux use?
▾
✦ বাংলা উত্তর

MLQ: একাধিক Queue, প্রতিটির আলাদা অগ্রাধিকার। প্রসেস একবার Queue-এ গেলে আর নড়তে পারে না।

MLFQ: প্রসেস Queue-এর মধ্যে চলাচল করতে পারে। বেশি CPU ব্যবহারে নামানো হয়, বেশি অপেক্ষায় তোলা হয় (Aging)। সবচেয়ে নমনীয়।

Windows: MLFQ (32 স্তর)। Linux: CFS (Completely Fair Scheduler) — Red-Black Tree ডেটা স্ট্রাকচার।
✦ English Answer

MLQ: Multiple queues each with their own priority and algorithm. Processes are permanently assigned to one queue — no movement allowed.

MLFQ: Processes can move between queues. Too much CPU usage → demoted. Too long waiting → promoted (Aging). Most flexible and realistic.

Windows uses MLFQ (32 priority levels). Linux uses CFS (Completely Fair Scheduler) with a Red-Black Tree data structure.

ডেডলক ডিটেকশন ও অ্যাভয়ডেন্স
Deadlock Detection and Avoidance
Q
বাং ডেডলক কী? ডেডলকের চারটি আবশ্যিক শর্ত কী কী?
EN What is Deadlock? What are its four necessary conditions?
▾
✦ বাংলা উত্তর

ডেডলক হলো এমন পরিস্থিতি যেখানে একটি প্রসেস অপেক্ষার অবস্থায় আটকে যায় কারণ যে রিসোর্সটি সে চাইছে সেটি অন্য একটি অপেক্ষারত প্রসেস ধরে আছে।

  1. Mutual Exclusion: রিসোর্স একবারে একটি প্রসেসই ব্যবহার করতে পারে।
  2. Hold and Wait: প্রসেস একটি রিসোর্স ধরে রেখে আরেকটির জন্য অপেক্ষা করছে।
  3. No Preemption: প্রসেস নিজে থেকে না ছাড়লে রিসোর্স কেড়ে নেওয়া যায় না।
  4. Circular Wait: P1→R1→P2→R2→P1 — একটি চক্র তৈরি হয়।
এই চারটি শর্ত একসাথে থাকলেই ডেডলক সম্ভব।
✦ English Answer

Deadlock occurs when a process enters a waiting state because a requested resource is held by another waiting process, which in turn is waiting for another resource.

  1. Mutual Exclusion: Only one process can use a resource at a time.
  2. Hold and Wait: A process holds one resource while waiting for another.
  3. No Preemption: A resource cannot be taken away unless the process releases it voluntarily.
  4. Circular Wait: P1→R1→P2→R2→P1 — a circular chain of waiting.
All four conditions must hold simultaneously for deadlock to occur.
Q
বাং ডেডলক হ্যান্ডলিংয়ের চারটি পদ্ধতি কী কী?
EN What are the four methods of handling Deadlock?
▾
✦ বাংলা উত্তর
  1. Deadlock Ignorance: সিস্টেম ডেডলক উপেক্ষা করে।
  2. Deadlock Prevention: সিস্টেম এমনভাবে ডিজাইন যাতে ডেডলক হতেই না পারে।
  3. Deadlock Avoidance: আগে থেকে পরীক্ষা করে ডেডলক হবে কি না দেখা হয়।
  4. Deadlock Detection: ডেডলক হতে দেওয়া হয়, তারপর শনাক্ত করে ঠিক করা হয়।
✦ English Answer
  1. Deadlock Ignorance: The system simply ignores deadlocks.
  2. Deadlock Prevention: The system is designed so deadlock can never occur.
  3. Deadlock Avoidance: The system checks in advance whether a deadlock might occur before granting resources.
  4. Deadlock Detection: Deadlock is allowed to occur, then detected and fixed.
Q
বাং Banker's Algorithm কীভাবে কাজ করে?
EN How does the Banker's Algorithm work?
▾
✦ বাংলা উত্তর

Banker's Algorithm হলো ডেডলক অ্যাভয়ডেন্সের একটি কৌশল। ব্যাংক যেমন নিশ্চিত করে সব গ্রাহকের চাহিদা মেটাতে পারবে তবেই ঋণ দেয় — ঠিক তেমনি।

  • প্রতিটি প্রসেসের সর্বোচ্চ রিসোর্স আগেই ঘোষণা করতে হয়।
  • Current Need = Max Need − Allocated Resource
  • "Safe State"-এ থাকলেই রিসোর্স বরাদ্দ দেওয়া হয়।
Safe State = ডেডলক হবে না। Unsafe State = ডেডলকের সম্ভাবনা।
✦ English Answer

The Banker's Algorithm is a deadlock avoidance strategy — named because it mimics how a bank ensures it can satisfy all customers' needs before lending money.

  • Each process must declare its maximum resource needs upfront.
  • Current Need = Max Need − Allocated Resource
  • Resources are granted only if the system remains in a "Safe State".
Safe State = no deadlock will occur. Unsafe State = deadlock may occur.
Q
বাং Resource Allocation Graph (RAG) কী? ডেডলক শনাক্তে কীভাবে ব্যবহৃত হয়?
EN What is a Resource Allocation Graph (RAG)? How is it used to detect deadlocks?
▾
✦ বাংলা উত্তর

RAG হলো একটি চিত্র যেখানে কোন প্রসেস কোন রিসোর্স ধরে আছে এবং কোনটির জন্য অপেক্ষা করছে তা দেখানো হয়।

  • Allocation Edge (R→P): রিসোর্স R, প্রসেস P-কে বরাদ্দ।
  • Request Edge (P→R): প্রসেস P, রিসোর্স R চাইছে।
গ্রাফে Cycle থাকলে → Single Instance-এ সবসময় Deadlock। Multiple Instance-এ হতেও পারে, নাও পারে।
✦ English Answer

A RAG is a visual representation showing which processes hold which resources and which processes are waiting for them.

  • Allocation Edge (R→P): Resource R is allocated to Process P.
  • Request Edge (P→R): Process P is requesting Resource R.
If a cycle exists → Deadlock is certain for single-instance resources. For multiple-instance resources, a cycle indicates a possibility but not certainty.
Q
বাং ডেডলক Avoidance ও Detection-এর মধ্যে পার্থক্য কী?
EN Compare Deadlock Avoidance and Deadlock Detection.
▾
✦ বাংলা উত্তর
  • Avoidance: প্রতিটি রিকোয়েস্টের আগে Safe State পরীক্ষা করে। প্রসেসকে সর্বোচ্চ চাহিদা আগেই জানাতে হয়। নিচু রিসোর্স ব্যবহার। নিরাপত্তামূলক সিস্টেমের জন্য।
  • Detection: প্রসেসকে স্বাধীনভাবে রিসোর্স নিতে দেয়। পর্যায়ক্রমে ডেডলক পরীক্ষা করে। বেশি রিসোর্স ব্যবহার। আধুনিক OS-এ বাস্তবসম্মত।
Avoidance = "প্রতিকারের চেয়ে প্রতিরোধ উত্তম" | Detection = "আশাবাদী পদ্ধতি"।
✦ English Answer
  • Avoidance (Proactive): Checks every request before granting to ensure Safe State. Processes must declare max needs in advance. Lower utilization. Best for safety-critical systems.
  • Detection (Reactive): Lets processes take resources freely. Periodically checks for deadlocks. Higher utilization. More practical for modern general-purpose OS.
Avoidance = "Prevention is better than cure" | Detection = "Optimistic approach".
Q
বাং ডেডলক রিকভারির পদ্ধতিগুলো কী?
EN What are the methods of Deadlock Recovery?
▾
✦ বাংলা উত্তর

১. প্রসেস টার্মিনেশন:

  • সব ডেডলক্ড প্রসেস একসাথে বন্ধ — নিশ্চিতভাবে ডেডলক ভাঙে, সব অগ্রগতি হারায়।
  • একটি একটি করে বন্ধ ও পুনরায় পরীক্ষা — কিছু প্রসেস রক্ষা পায়।

২. রিসোর্স Preemption: Victim প্রসেস বেছে রিসোর্স কেড়ে নেওয়া। প্রসেসকে Rollback করে পূর্বের Safe State-এ ফেরানো। Starvation-এর ঝুঁকি।

✦ English Answer

1. Process Termination:

  • Abort all deadlocked processes — guaranteed to break deadlock but all progress is lost.
  • Abort one at a time and re-run detection — saves some processes.

2. Resource Preemption: Select a victim process and take its resource. Roll the process back to a previous Safe State checkpoint. Risk of Starvation if the same process is always chosen as victim.
